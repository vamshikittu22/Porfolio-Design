**Create ThemeContext (src/contexts/ThemeContext.tsx):**
- Theme modes: `'light' | 'dark' | 'system'`
- State: `themeMode` (user preference), `resolvedTheme` (actual applied theme)
- localStorage key: 'portfolio-theme-mode'
- On mount: read from localStorage, default to 'system'
- System detection: `window.matchMedia('(prefers-color-scheme: dark)')`
- Listen to system preference changes with matchMedia.addEventListener
- Apply theme by adding/removing 'dark' class on document.documentElement
- Export: ThemeProvider, useTheme hook

**Create ThemeToggle component (src/components/ui/ThemeToggle.tsx):**
- 3-state toggle: Light → Dark → System (cycles through)
- Icons: ☀️ (light), 🌙 (dark), 🖥️ (system)
- Glass morphism button matching ChapterSidebar style
- Show current mode with icon + label on hover
- Framer Motion transitions for icon changes
- Match existing navigation button sizing (~44px)

**Integrate ThemeToggle into ChapterSidebar:**
- Add ThemeToggle button at BOTTOM of sidebar (below all chapters)
- Position: fixed left-6, bottom-6 (below chapter buttons)
- Should appear on desktop only (md:flex hidden on mobile)
- For mobile: Add to ChapterBottomSheet header area (next to close button)

**Migration from existing isDarkMode:**
- App.tsx currently has isDarkMode state and localStorage
- ThemeContext replaces this logic
- Preserve existing theme from localStorage if present
- Remove isDarkMode state from App.tsx in favor of useTheme hook

Why ThemeContext:
- Centralized theme management vs scattered state
- Supports system preference listening + manual override
- Reusable across all components
- Persistent user preference
- Decouples theme from App.tsx state
