Implementation based on research (02-RESEARCH.md lines 272-323):

**Component structure:**

```tsx
import { motion } from 'motion/react';
import { useNavigation } from '@/contexts/NavigationContext';
import { useReducedMotion } from '@/hooks/useReducedMotion';
import type { Chapter } from '@/types/chapters';

interface ChapterCardProps {
  chapter: Chapter;
}

export function ChapterCard({ chapter }: ChapterCardProps) {
  const { navigateToChapter } = useNavigation();
  const prefersReducedMotion = useReducedMotion();

  const handleNavigate = () => {
    navigateToChapter(chapter.id, 'jump');  // Use 'jump' to reset scroll to top
  };

  return (
    <motion.article
      className="glass-card chapter-card"
      whileHover={prefersReducedMotion ? {} : { scale: 1.02, y: -4 }}
      whileTap={prefersReducedMotion ? {} : { scale: 0.98 }}
      onTap={handleNavigate}
      role="button"
      tabIndex={0}
      onKeyDown={(e) => {
        if (e.key === 'Enter' || e.key === ' ') {
          e.preventDefault();
          handleNavigate();
        }
      }}
      aria-label={`Navigate to ${chapter.title}`}
      style={{
        cursor: 'pointer',
        minHeight: '200px',  // Ensures adequate touch target
      }}
    >
      <div className="card-content">
        {/* Chapter number indicator */}
        <div className="card-number">
          <span className="text-4xl font-bold text-white/40">
            {chapter.number.toString().padStart(2, '0')}
          </span>
        </div>
        
        {/* Chapter icon placeholder - icons deferred per Phase 1 note */}
        {/* TODO: Replace with actual icon when chapter icons created */}
        <div className="card-icon" aria-hidden="true">
          <div className="w-16 h-16 rounded-full bg-white/10 flex items-center justify-center">
            <span className="text-2xl">{chapter.number}</span>
          </div>
        </div>
        
        {/* Chapter title */}
        <h2 className="card-title text-2xl font-semibold mt-4">
          {chapter.title}
        </h2>
        
        {/* Chapter description */}
        <p className="card-description text-sm text-white/70 mt-2">
          {chapter.description}
        </p>
      </div>
    </motion.article>
  );
}
```

**Key implementation details:**

1. **Navigation:** Use `navigateToChapter(chapter.id, 'jump')` with 'jump' type to reset scroll to top when entering chapter (per Phase 1 pattern)

2. **Accessibility:**
   - Use `role="button"` for semantic button behavior
   - Add `tabIndex={0}` for keyboard focus
   - Handle Enter and Space key for keyboard activation
   - Use `aria-label` for screen reader context
   - Ensure `minHeight: 200px` for comfortable touch target (exceeds 44px minimum)

3. **Animations:**
   - Use Framer Motion's `whileHover` and `whileTap` gestures (research pattern)
   - Hover: slight lift (scale 1.02, y -4px)
   - Tap: slight press (scale 0.98)
   - Respect reduced motion by passing empty objects when preference enabled

4. **Icon placeholder:**
   - Use number-only UI consistent with Phase 1 decision (STATE.md line 74)
   - Add TODO comment for future icon integration

**DO NOT:**
- Use `onClick` instead of `onTap` with Framer Motion (breaks gesture handling - research line 265)
- Animate on mount without reduced motion check (research anti-pattern line 172)
- Use `index` as key when mapping (should use chapter.id - research line 173)

```css
/**
 * Glass Morphism Card Styling
 * Extends existing glass UI patterns for chapter cards
 * Research: 02-RESEARCH.md Pattern 3
 */

.glass-card {
  /* Base glass morphism effect */
  background: rgba(255, 255, 255, 0.1);
  border: 1px solid rgba(255, 255, 255, 0.2);
  border-radius: 16px;
  box-shadow: 
    0 8px 32px rgba(0, 0, 0, 0.1),
    0 1px 2px rgba(0, 0, 0, 0.05);
  
  /* Transition for smooth hover effects */
  transition: all 0.3s ease;
}

/* Backdrop blur with progressive enhancement */
@supports (backdrop-filter: blur(10px)) {
  .glass-card {
    backdrop-filter: blur(10px);
  }
}

/* Fallback for prefers-reduced-motion */
@media (prefers-reduced-motion: reduce) {
  .glass-card {
    backdrop-filter: none;
    background: rgba(255, 255, 255, 0.15); /* Slightly more opaque */
    transition: none;
  }
}

/* Hover state - enhanced elevation and border */
.glass-card:hover {
  border-color: rgba(255, 255, 255, 0.3);
  box-shadow: 
    0 12px 48px rgba(0, 0, 0, 0.15),
    0 2px 4px rgba(0, 0, 0, 0.08);
}

/* Active/tap state - reduced elevation */
.glass-card:active {
  box-shadow: 
    0 4px 16px rgba(0, 0, 0, 0.08),
    0 1px 2px rgba(0, 0, 0, 0.05);
}

/* Focus state for keyboard navigation */
.glass-card:focus-visible {
  outline: 2px solid rgba(255, 255, 255, 0.5);
  outline-offset: 2px;
}

/* Chapter card specific layout */
.chapter-card {
  padding: 1.5rem; /* 24px */
  display: flex;
  flex-direction: column;
  justify-content: space-between;
}

.chapter-card .card-content {
  display: flex;
  flex-direction: column;
  gap: 0.5rem;
}

.chapter-card .card-number {
  position: absolute;
  top: 1rem;
  right: 1rem;
  opacity: 0.3;
}

.chapter-card .card-icon {
  margin-bottom: 0.75rem;
}

.chapter-card .card-title {
  font-weight: 600;
  line-height: 1.2;
}

.chapter-card .card-description {
  line-height: 1.5;
  opacity: 0.8;
}

/* Ensure adequate touch targets on mobile */
@media (max-width: 640px) {
  .chapter-card {
    min-height: 180px; /* Comfortable tap target */
    padding: 1rem;
  }
}
```

**Key patterns:**

1. **Progressive enhancement:** Use `@supports` for backdrop-filter to ensure fallback (research line 416)

2. **Reduced motion support:** Disable backdrop-filter and transitions when user prefers reduced motion (research lines 229-234)

3. **Accessibility:** Add `:focus-visible` state for keyboard navigation

4. **Performance:** Use `backdrop-filter` conditionally to avoid jank on low-end devices (research concern line 217)

5. **Responsive touch targets:** Ensure min-height on mobile meets WCAG 2.5.5 (44px minimum, using 180px for comfort)

**DO NOT:**
- Use backdrop-filter without `@supports` check (breaks on unsupported browsers)
- Animate backdrop-filter value (performance issue - research line 217)
- Set hard-coded heights that prevent content flexibility
