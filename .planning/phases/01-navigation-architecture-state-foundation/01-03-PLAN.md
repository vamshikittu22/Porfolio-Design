Create hook to detect user's motion preference for accessibility:

**useReducedMotion hook**:

1. **Return type**: boolean (true if reduced motion preferred)

2. **Implementation**:
   - Use window.matchMedia('(prefers-reduced-motion: reduce)')
   - useState to track preference
   - useEffect to add change listener (user can toggle during session)

   - Check if window exists (SSR compatibility, though Vite is client-only)
   - Default to false if matchMedia not available

4. **Example usage**:
```typescript

const transition = prefersReducedMotion 
  ? { duration: 0 } 
  : { duration: 0.3, ease: 'easeInOut' };
```

<task type="auto">
  <name>Create chapter transition wrapper</name>
  <files>src/components/transitions/ChapterTransition.tsx</files>
  <action>
Create Framer Motion wrapper for chapter content transitions:

**ChapterTransition component**:

1. **Props**:
   - children: React.ReactNode
   - chapterId: ChapterId | null (used as AnimatePresence key)

2. **Hooks**:
   - useNavigation() to access navigationType, isTransitioning
   - useReducedMotion() to detect motion preference

3. **Scroll management** (useEffect):
   - Watch currentChapter changes
   - If navigationType === 'jump': window.scrollTo({ top: 0, behavior: 'smooth' })
   - If navigationType === 'sequential': Do nothing (preserve scroll)
   - Runs on currentChapter change

4. **Animation config**:
   - Use Framer Motion's `<AnimatePresence mode="wait">`
   - Wrap children in `<motion.div>` with key={chapterId}
   - Animations:
     - **initial**: { opacity: 0 }
     - **animate**: { opacity: 1 }
     - **exit**: { opacity: 0 }
   - **transition**: 
     - If prefersReducedMotion: { duration: 0 }
     - Else: { duration: 0.3, ease: 'easeInOut' }

5. **Prevent interaction during transition**:
   - If isTransitioning: Add pointer-events-none class
   - Prevents double-clicks on navigation during animation

6. **Layout**:
   - No styling on wrapper (transparent pass-through)
   - Children render normally within fade container

**Example structure**:
```typescript
<AnimatePresence mode="wait">
  <motion.div
    key={chapterId || 'landing'}
    initial={{ opacity: 0 }}
    animate={{ opacity: 1 }}
    exit={{ opacity: 0 }}
    transition={transition}
    className={isTransitioning ? 'pointer-events-none' : ''}
  >
    {children}
  </motion.div>
</AnimatePresence>
```

**IMPORTANT**: Use the `currentChapter` from NavigationContext as the key, NOT a prop. Access via useNavigation() inside component.
