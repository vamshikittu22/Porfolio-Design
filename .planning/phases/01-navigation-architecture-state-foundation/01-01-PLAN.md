  <name>Create NavigationContext with hash synchronization</name>

  <action>
Create React Context for navigation state management with bidirectional hash sync:

**NavigationProvider component**:

1. **State initialization**:
   - currentChapter: Read from window.location.hash on mount (strip '#', validate against CHAPTERS)
   - Default to null if no hash or invalid hash
   - isMenuOpen: false
   - isTransitioning: false
   - navigationType: 'jump'

2. **Hash change listener** (useEffect):
   - Listen to 'hashchange' event
   - Parse hash, update currentChapter state
   - Validate hash against CHAPTERS registry
   - If invalid hash, clear to null (show landing page)

3. **Navigation methods**:
   - **navigateToChapter(chapterId: ChapterId, type: NavigationType = 'jump')**:
     - Set isTransitioning to true
     - Update window.location.hash to `#${chapterId}`
     - Update navigationType state
     - Close menu (isMenuOpen = false)
     - After 300ms, set isTransitioning to false
   
   - **toggleMenu()**: Toggle isMenuOpen
   - **closeMenu()**: Set isMenuOpen to false
   - **openMenu()**: Set isMenuOpen to true

4. **Context value**: Memoize context object with useMemo to prevent unnecessary rerenders

5. **useNavigation hook**: 
   - Custom hook that calls useContext
   - Throws error if used outside NavigationProvider
   - Returns typed context value

**Implementation notes**:
- Use hashchange event, not popstate (for hash-based routing)
- Validate hash on every change to prevent broken states
- Export both NavigationProvider and useNavigation hook
- Follow React 19 patterns (no FC type, use function components)

Reference existing state patterns from app/App.tsx lines 28-37 for React 19 conventions.
