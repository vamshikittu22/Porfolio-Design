### Task 1: Configure Manual Chunking Strategy

**File:** `vite.config.ts`

Split large dependencies into separate vendor chunks for better caching and parallel loading.

**Changes:**

```typescript
import path from 'path';
import { defineConfig, loadEnv } from 'vite';
import react from '@vitejs/plugin-react';

export default defineConfig(({ mode }) => {
    const env = loadEnv(mode, '.', '');
    return {
      server: {
        port: 3000,
        host: '0.0.0.0',
      },
      plugins: [react()],
      define: {
        'process.env.API_KEY': JSON.stringify(env.GEMINI_API_KEY),
        'process.env.GEMINI_API_KEY': JSON.stringify(env.GEMINI_API_KEY)
      },
      resolve: {
        alias: {
          '@': path.resolve(__dirname, '.'),
        }
      },
      build: {
        // Increase chunk size warning limit to 600kB to focus on real issues
        chunkSizeWarningLimit: 600,
        
        rollupOptions: {
          output: {
            manualChunks: (id) => {
              // React core libraries
              if (id.includes('node_modules/react') || id.includes('node_modules/react-dom')) {
                return 'vendor-react';
              }
              
              // Framer Motion animation library (large)
              if (id.includes('node_modules/framer-motion') || id.includes('node_modules/motion')) {
                return 'vendor-motion';
              }
              
              // Google GenAI SDK (used only in ChatAssistant)
              if (id.includes('node_modules/@google/genai')) {
                return 'vendor-genai';
              }
              
              // Chapter components - lazy load by default
              if (id.includes('src/components/chapters/')) {
                const match = id.match(/chapters\/([^\/]+)/);
                if (match) {
                  return `chapter-${match[1].toLowerCase()}`;
                }
              }
              
              // All other node_modules
              if (id.includes('node_modules')) {
                return 'vendor-libs';
              }
            },
          },
        },
      },
    };
});
```

**Expected bundle breakdown after changes:**
- `vendor-react.js`: ~140kB (React 19 + ReactDOM)
- `vendor-motion.js`: ~120kB (Framer Motion + motion)
- `vendor-genai.js`: ~80kB (Google GenAI SDK)
- `vendor-libs.js`: ~40kB (other dependencies)
- `index.js`: ~150-200kB (application code)
- Chapter chunks: 8-20kB each (lazy loaded)
- Section chunks: 8-20kB each (already lazy loaded)

**Acceptance:**
- vite.config.ts updated with manualChunks configuration
- Build produces separate vendor-*.js chunks
- Main bundle reduced to < 300kB
- No breaking changes in functionality

### Task 2: Fix CSS Import Order

**File:** `src/index.css`

Fix Vite warnings about @import order by moving custom imports before Tailwind directives.

**Current (incorrect):**
```css
@tailwind base;
@tailwind components;
@tailwind utilities;

/* Import custom styles */
@import './styles/glass-cards.css';
@import './styles/glass-morphism.css';
@import './styles/landing.css';
@import './styles/chapter.css';
```

**Corrected:**
```css
/* Import custom styles BEFORE Tailwind directives */
@import './styles/glass-cards.css';
@import './styles/glass-morphism.css';
@import './styles/landing.css';
@import './styles/chapter.css';

@tailwind base;
@tailwind components;
@tailwind utilities;
```

**Why this matters:**
- CSS @import must precede all other statements (CSS spec requirement)
- Prevents build warnings
- Ensures custom styles can reference Tailwind theme values correctly

**Acceptance:**
- CSS imports moved above @tailwind directives
- Build completes without CSS import warnings
- No visual regression in styling

### Task 3: Verify Tree-Shaking for Framer Motion

**File:** Check import patterns across codebase

Ensure Framer Motion is imported from tree-shakeable `motion` package (v12.x), not `framer-motion`.

**Verification:**

1. Run search for framer-motion imports:

### Task 5: Enable Compression in Preview Server

**File:** `vite.config.ts`

Enable gzip/brotli compression for preview server to simulate production environment.

**Changes:**

```typescript
import { defineConfig, loadEnv } from 'vite';
import react from '@vitejs/plugin-react';
import { visualizer } from 'rollup-plugin-visualizer';
import compression from 'vite-plugin-compression';

export default defineConfig(({ mode }) => {
    const env = loadEnv(mode, '.', '');
    return {
      server: {
        port: 3000,
        host: '0.0.0.0',
      },
      plugins: [
        react(),
        visualizer({
          filename: './dist/stats.html',
          open: false,
          gzipSize: true,
          brotliSize: true,
        }),
        // Brotli compression for preview builds
        compression({
          algorithm: 'brotliCompress',
          ext: '.br',
        }),
        // Gzip compression for preview builds
        compression({
          algorithm: 'gzip',
          ext: '.gz',
        }),
      ],
      // ... rest of config
    };
});
```

**Install dependency:**
```bash
npm install --save-dev vite-plugin-compression
```

**Acceptance:**
- vite-plugin-compression installed
- Brotli and gzip compression enabled
- Preview server serves compressed assets
- Can measure realistic production sizes
