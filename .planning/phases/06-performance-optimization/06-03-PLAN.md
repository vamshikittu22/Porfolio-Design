### Task 1: Dynamic Import for ChatAssistant

**File:** `src/App.tsx`

Lazy load ChatAssistant component only when user clicks the launcher button.

**Current (eager loading):**
```typescript
import ChatAssistant from './components/layout/ChatAssistant/ChatAssistant';

function App() {
  return (
    <NavigationProvider>
      <ThemeProvider>
        {/* ... */}
        <ChatAssistant />
      </ThemeProvider>
    </NavigationProvider>
  );
}
```

**Optimized (lazy loading):**
```typescript
import { lazy, Suspense, useState } from 'react';

// Lazy load ChatAssistant (includes Google GenAI SDK)
const ChatAssistant = lazy(() => import('./components/layout/ChatAssistant/ChatAssistant'));

function App() {
  const [isChatReady, setIsChatReady] = useState(false);

  return (
    <NavigationProvider>
      <ThemeProvider>
        {/* ... navigation and content ... */}
        
        {/* Chat launcher button - always visible */}
        {!isChatReady && (
          <button
            onClick={() => setIsChatReady(true)}
            className="fixed bottom-8 right-8 z-[200] w-20 h-20 rounded-[28px] bg-t-bg-el/90 backdrop-blur-2xl border border-t-border/50 flex items-center justify-center shadow-xl"
            aria-label="Open portfolio assistant"
          >
            <svg className="w-6 h-6" fill="none" stroke="currentColor" viewBox="0 0 24 24" strokeWidth={2}>
              <path d="M13 10V3L4 14h7v7l9-11h-7z" />
            </svg>
          </button>
        )}
        
        {/* ChatAssistant - loaded on demand */}
        {isChatReady && (
          <Suspense fallback={null}>
            <ChatAssistant />
          </Suspense>
        )}
      </ThemeProvider>
    </NavigationProvider>
  );
}
```

**Alternative approach: Load on first interaction**
```typescript
const ChatAssistant = lazy(() => import('./components/layout/ChatAssistant/ChatAssistant'));

function App() {
  const [shouldLoadChat, setShouldLoadChat] = useState(false);

  useEffect(() => {
    // Preload chat on idle (after 3 seconds of inactivity)
    const idleTimer = setTimeout(() => {
      setShouldLoadChat(true);
    }, 3000);

    // Or preload on first scroll/interaction
    const handleInteraction = () => {
      setShouldLoadChat(true);
      window.removeEventListener('scroll', handleInteraction);
      window.removeEventListener('click', handleInteraction);
    };

    window.addEventListener('scroll', handleInteraction, { once: true, passive: true });
    window.addEventListener('click', handleInteraction, { once: true });

    return () => {
      clearTimeout(idleTimer);
      window.removeEventListener('scroll', handleInteraction);
      window.removeEventListener('click', handleInteraction);
    };
  }, []);

  return (
    <NavigationProvider>
      <ThemeProvider>
        {/* ... */}
        
        {shouldLoadChat && (
          <Suspense fallback={null}>
            <ChatAssistant />
          </Suspense>
        )}
      </ThemeProvider>
    </NavigationProvider>
  );
}
```

**Decision:** Use on-click loading for maximum savings, or idle loading for better UX.

**Acceptance:**
- ChatAssistant imported via React.lazy()
- Component only loads when user opens chat
- Fallback UI provided during loading (Suspense)
- ~100kB savings on initial bundle

### Task 2: Optimize Font Loading

**File:** `index.html` + `src/index.css`

Add font-display: swap and preconnect hints for faster font loading.

**Current (if using Google Fonts):**
```html
<!-- index.html -->
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;700&display=swap" rel="stylesheet">
```

**Optimized:**
```html
<!-- index.html -->
<head>
  <!-- ... -->
  
  <!-- Preconnect to Google Fonts for faster DNS resolution -->
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
  
  <!-- Font loading with font-display: swap -->
  <link 
    href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;700;800;900&display=swap" 
    rel="stylesheet"
  >
</head>
```

**Or use self-hosted fonts:**

**File:** `public/fonts/inter.woff2` (download from Google Fonts)

**File:** `src/index.css`
```css
/* Self-hosted fonts for better performance */
@font-face {
  font-family: 'Inter';
  font-style: normal;
  font-weight: 400;
  font-display: swap;
  src: url('/fonts/inter-regular.woff2') format('woff2');
}

@font-face {
  font-family: 'Inter';
  font-style: normal;
  font-weight: 700;
  font-display: swap;
  src: url('/fonts/inter-bold.woff2') format('woff2');
}

/* ... other weights */
```

**Benefits:**
- `font-display: swap` → Shows fallback font immediately, swaps when custom font loads
- Preconnect → Reduces DNS lookup time for Google Fonts
- Self-hosted → Eliminates third-party dependency, better caching

**Decision:** Use preconnect + font-display: swap for simplicity, or self-host for maximum control.

**Acceptance:**
- font-display: swap applied to all fonts
- Preconnect hints added for Google Fonts (if used)
- No FOIT (Flash of Invisible Text) during loading
- Fonts load asynchronously without blocking render

### Task 4: Optimize Images with Lazy Loading

**File:** Review all image usages in codebase

Ensure all below-the-fold images use loading="lazy" attribute.

**Search for image tags:**
```bash
grep -r "<img" src/components --include="*.tsx"
grep -r "background-image" src/ --include="*.css"
```

**Update image tags:**

**Before:**
```typescript
<img src="/icons/github.svg" alt="GitHub" />
```

**After:**
```typescript
<img 
  src="/icons/github.svg" 
  alt="GitHub"
  loading="lazy"
  decoding="async"
/>
```

**Exceptions (do NOT lazy load):**
- Hero section images (above the fold)
- Logo images (visible on load)
- Critical UI icons

**For background images in CSS:**
```css
/* Use will-change hint for animated backgrounds */
.hero-background {
  background-image: url('/images/hero-bg.jpg');
  background-size: cover;
  will-change: transform;
}
```

**Acceptance:**
- All below-the-fold images use loading="lazy"
- Above-the-fold images load eagerly (no lazy loading)
- decoding="async" added for non-blocking decode
- No broken images or layout shifts

### Task 5: Add Service Worker for Offline Support (Optional)

**File:** `vite.config.ts` + `public/sw.js`

Configure Vite PWA plugin for offline caching and faster repeat visits.

**Install plugin:**
```bash
npm install --save-dev vite-plugin-pwa
```

**Update vite.config.ts:**
```typescript
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import { VitePWA } from 'vite-plugin-pwa';

export default defineConfig({
  plugins: [
    react(),
    VitePWA({
      registerType: 'autoUpdate',
      includeAssets: ['favicon.ico', 'robots.txt', 'icons/**/*'],
      manifest: {
        name: 'Vamshi Kothapalli Portfolio',
        short_name: 'VK Portfolio',
        description: 'Portfolio showcasing full-stack development work',
        theme_color: '#3b82f6',
        background_color: '#0a0a0a',
        display: 'standalone',
        icons: [
          {
            src: '/icons/icon-192.png',
            sizes: '192x192',
            type: 'image/png'
          },
          {
            src: '/icons/icon-512.png',
            sizes: '512x512',
            type: 'image/png'
          }
        ]
      },
      workbox: {
        globPatterns: ['**/*.{js,css,html,ico,png,svg,woff2}'],
        runtimeCaching: [
          {
            urlPattern: /^https:\/\/fonts\.googleapis\.com\/.*/i,
            handler: 'CacheFirst',
            options: {
              cacheName: 'google-fonts-cache',
              expiration: {
                maxEntries: 10,
                maxAgeSeconds: 60 * 60 * 24 * 365 // 1 year
              },
              cacheableResponse: {
                statuses: [0, 200]
              }
            }
          }
        ]
      }
    })
  ]
});
```

**Benefits:**
- Offline support (portfolio accessible without internet)
- Faster repeat visits (assets cached)
- PWA installable on mobile/desktop

**Tradeoffs:**
- Adds ~10kB to bundle (Workbox runtime)
- Requires HTTPS for production
- Cache invalidation complexity

**Decision:** Implement if time permits, defer to post-Phase 6 if needed.

**Acceptance:**
- Vite PWA plugin installed and configured
- Service worker registers on load
- Offline mode works (test by disconnecting internet)
- Lighthouse PWA audit passes

### Task 6: Extract Critical CSS (Advanced)

**File:** `vite.config.ts` + build process

Inline critical above-the-fold CSS in HTML to eliminate render-blocking CSS.

**Install plugin:**
```bash
npm install --save-dev vite-plugin-critical
```

**Update vite.config.ts:**
```typescript
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import critical from 'vite-plugin-critical';

export default defineConfig({
  plugins: [
    react(),
    critical({
      pages: ['/', '/#01-introduction'],
      width: 1440,
      height: 900,
      inline: true,
      minify: true
    })
  ]
});
```

**How it works:**
1. Plugin renders pages in headless browser
2. Extracts CSS used above the fold
3. Inlines critical CSS in <head>
4. Defers full CSS file with media="print" hack

**Benefits:**
- Eliminates render-blocking CSS
- Faster First Contentful Paint
- Improves Lighthouse score by 5-10 points

**Tradeoffs:**
- Increases HTML file size
- Build time increases
- May cause flash of unstyled content (FOUC) if not tuned

**Decision:** Implement if landing page FCP > 1.5s after previous optimizations.

**Acceptance:**
- Critical CSS plugin configured
- Landing page CSS inlined in HTML
- Full CSS loads asynchronously
- No FOUC on page load
